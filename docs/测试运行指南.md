## 环境准备

### 必需软件

| 软件 | 版本 | 下载 |
|------|------|------|
| Python | 3.11+ | https://www.python.org/downloads/ |
| Node.js | 18+ | https://nodejs.org/ |
| Ollama | 最新 | https://ollama.com/download |
| Docker Desktop | 最新（Docker部署时） | https://www.docker.com/products/docker-desktop |

### 必需服务

| 服务 | 默认端口 | 启动方式 |
|------|----------|----------|
| Ollama | 11434 | `ollama serve` |
| Qdrant | 6333 | Docker 或后端自动创建本地存储 |
| MySQL/PostgreSQL | 3306/5432 | 确保业务数据库可连接 |

### 下载模型

```bash
ollama pull qwen3:14b
ollama pull qwen3.5:9b
```

## 本地启动

### 1. 配置环境

```bash
copy .env.example .env
notepad .env
```

关键配置项：
```bash
DB_TYPE=mysql
DB_HOST=localhost
DB_PORT=3306
DB_NAME=desk_management
DB_USER=root
DB_PASSWORD=your_password
CHAT_MODEL=qwen3.5:9b
```

### 2. 安装依赖

```bash
pip install -r requirements.txt
```

### 3. 启动后端

```bash
uvicorn agent_backend.main:app --reload --host 0.0.0.0 --port 8000
```

验证：`http://localhost:8000/api/v1/health` 返回 `{"status":"ok"}`

### 4. 启动前端

```bash
cd agent_frontend
npm install
npm run dev
```

验证：`http://localhost:5173` 显示前端界面

### 5. 功能验证

- 在前端输入"你好"验证对话
- 输入"当前有多少部门"验证 SQL 查询
- 输入"部门下机器数量最多的前十部门，以及折线图"验证图表+导出

## Docker 部署

### 一键部署

```bash
docker\deploy.bat       # Windows
./docker/deploy.sh      # Linux/Mac
```

### 手动部署

```bash
docker-compose build
docker-compose up -d
```

### 访问服务

| 服务 | 地址 |
|------|------|
| 前端 | http://localhost |
| 后端 API 文档 | http://localhost:8000/docs |
| Qdrant 控制台 | http://localhost:6333/dashboard |

### 常用命令

```bash
docker-compose logs -f          # 查看日志
docker-compose logs -f backend  # 查看后端日志
docker-compose down             # 停止服务
docker-compose restart          # 重启服务
```

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| ModuleNotFoundError | `pip install -r requirements.txt` |
| Ollama 连接失败 | 检查 `ollama list`，运行 `ollama serve` |
| Qdrant 连接失败 | `docker run -d -p 6333:6333 qdrant/qdrant` |
| 前端无法连接后端 | 检查后端是否运行，检查 `vite.config.js` 代理设置 |
| 数据库连接失败 | 检查 `.env` 中的数据库配置，运行 `python scripts/测试数据库连接.py` |
| Docker 构建失败 | `docker system prune -a` 后 `docker-compose build --no-cache` |
