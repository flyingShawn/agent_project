# 配置文件完整说明

## 概述

所有配置都在项目根目录的 **`.env`** 文件中，无需修改代码即可完成所有配置。

从 `.env.example` 复制后修改：

```bash
# Windows
copy .env.example .env

# Linux/Mac
cp .env.example .env
```

---

## 一、大模型配置

### 1.1 LLM 服务地址

```bash
LLM_BASE_URL=http://localhost:11434/v1
```

**说明**：LLM 服务地址（OpenAI 兼容协议 `/v1/chat/completions`）

**可选后端**：

| 后端 | LLM_BASE_URL | 需要 API Key |
|------|-------------|-------------|
| 本地 Ollama | `http://localhost:11434/v1` | 否 |
| DeepSeek | `https://api.deepseek.com/v1` | 是 |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | 是 |

### 1.2 LLM API Key

```bash
LLM_API_KEY=
```

**说明**：本地 Ollama 留空即可，云端服务必填。

### 1.3 文本对话模型

```bash
CHAT_MODEL=qwen3.5:9b
```

**可选模型**：

| 模型名称 | 大小 | 适用场景 |
|---------|------|---------|
| qwen3.5:9b | 约4.7GB | 平衡性能，推荐使用 |
| qwen3:14b | 约9GB | 高配置机器，效果最好 |
| deepseek-chat | - | DeepSeek 云端 |
| qwen-plus | - | 通义千问云端 |

**下载模型**：
```bash
ollama pull qwen3.5:9b
```

### 1.4 视觉模型

```bash
VISION_MODEL=qwen3.5:9b
```

**说明**：用于图片理解和多模态 RAG 的模型

**可选模型**：

| 模型名称 | 大小 | 适用场景 |
|---------|------|---------|
| qwen2.5-vl:3b | 约2GB | 低配置机器 |
| qwen2.5-vl:7b | 约4.7GB | 推荐，平衡性能 |

### 1.5 云端兜底开关

```bash
ENABLE_CLOUD_FALLBACK=0
```

**说明**：0=不启用，1=启用。启用后当本地 Ollama 不可用时自动切换到云端 API。

## 二、数据库配置（只读）

### 2.1 数据库类型

```bash
DB_TYPE=mysql
```

**可选值**：`mysql` 或 `postgresql`

### 2.2 连接信息

**MySQL 配置**：
```bash
DB_HOST=localhost
DB_PORT=3306
DB_NAME=desk_management
DB_USER=root
DB_PASSWORD=your_password
```

**PostgreSQL 配置**：
```bash
DB_HOST=localhost
DB_PORT=5432
DB_NAME=desk_management
DB_USER=postgres
DB_PASSWORD=your_password
```

### 2.3 完整 URL 配置（可选）

```bash
DATABASE_URL=mysql+pymysql://root:password@localhost:3306/db?charset=utf8mb4
```

**优先级**：如果填写了 `DATABASE_URL`，将优先使用它，忽略单独的 DB_HOST/DB_PORT 等配置。

**格式**：
- MySQL: `mysql+pymysql://user:password@host:3306/db?charset=utf8mb4`
- PostgreSQL: `postgresql://user:password@host:5432/db`

**注意**：密码中的特殊字符需要 URL 编码。

## 三、RAG 文档问答配置

### 3.1 文档目录

```bash
RAG_DOCS_DIR=./data/desk-agent/docs
```

**支持格式**：PDF, Word (.docx), PowerPoint (.pptx), Excel (.xlsx), Markdown (.md), 纯文本 (.txt), 图片 (PNG/JPG/WebP)

### 3.2 向量数据库

```bash
RAG_QDRANT_URL=http://localhost:6333
RAG_QDRANT_PATH=./.qdrant_local
RAG_QDRANT_COLLECTION=desk_agent_docs
```

### 3.3 嵌入模型

```bash
RAG_EMBEDDING_MODEL=BAAI/bge-small-zh-v1.5
```

**可选模型**：
- `BAAI/bge-small-zh-v1.5`：中文小模型（推荐，约 100MB）
- `BAAI/bge-base-zh-v1.5`：中文基础模型
- `BAAI/bge-large-zh-v1.5`：中文大模型

### 3.4 检索参数

```bash
RAG_HYBRID_ALPHA=0.7      # 混合检索权重（越大向量权重越高）
RAG_TOP_K=5               # 返回文档数
RAG_CANDIDATE_K=30        # 候选文档数（用于重排序）
RAG_VECTOR_MIN_SCORE=0.5  # 最低相似度阈值（低于此值过滤）
```

## 七、前端配置

```bash
APP_NAME=阳途智能助手                    # 应用名称（标题栏）
APP_SUBTITLE=阳途智能助手为您服务         # 应用副标题
APP_WELCOME_TEXT=有什么我能帮您的呢？      # 欢迎语
APP_INPUT_PLACEHOLDER=给智能助手发消息     # 输入框占位文字
QUICK_OPTIONS=查看客户端在线状态,...       # 快捷选项（逗号分隔）
```

前端显示文案统一维护在仓库根目录 `.env` 的 `VITE_*` 配置项中：
- **本地开发 / 本地构建**：`agent_frontend/vite.config.js` 通过 `envDir` 读取根目录 `.env`
- **Docker 部署**：`docker-compose.yml` 从根目录 `.env` 读取同一批变量，`entrypoint.frontend.sh` 在容器启动时生成 `config.js`
- `agent_frontend/public/config.js` 只是占位文件，不再手动维护业务文案

## 八、YAML 配置文件

### schema_metadata.yaml

路径：`agent_backend/configs/schema_metadata.yaml`

定义数据库表结构、同义词、安全规则、权限控制和查询模板。

| 配置项 | 说明 |
|--------|------|
| `tables` | 数据库表结构定义（表名、字段、类型、注释） |
| `relationships` | 表间关联关系（用于 JOIN 推断） |
| `synonyms` | 字段同义词映射（帮助 LLM 理解用户意图） |
| `security` | 安全规则（受限表、禁止查询的敏感列） |
| `permissions` | 权限控制规则（按管理员限制可见部门/设备） |
| `query_patterns` | 预定义 SQL 模板（命中则绕过 LLM 直接返回） |
| `display_fields` | 不同查询场景的展示字段规范 |

### scheduled_tasks.yaml

路径：`agent_backend/configs/scheduled_tasks.yaml`

定义应用启动时自动加载的默认定时任务。支持 interval（固定间隔）和 cron（Cron 表达式）两种调度类型。应用启动时会自动与数据库中已有任务对比，实现配置热更新。

## 常见问题

**修改配置后不生效**
1. 确保修改的是 `.env` 文件，而不是 `.env.example`
2. 重启后端服务

**模型名称错误**
- 查看 Ollama 可用模型：`ollama list`
- 下载模型：`ollama pull qwen3.5:9b`

**数据库连接失败**
1. 数据库服务是否启动
2. IP 地址和端口是否正确
3. 用户名和密码是否正确
4. 防火墙是否开放端口

**Docker 容器内访问宿主机 Ollama**
- Windows/macOS：`LLM_BASE_URL=http://host.docker.internal:11434/v1`
- Linux：`LLM_BASE_URL=http://172.17.0.1:11434/v1`

## 最佳实践

1. **使用只读数据库账号**：为 AI 智能体创建专用的只读账号
2. **定期备份配置**：备份 `.env` 文件（注意保护密码）
3. **根据硬件选择模型**：不要盲目追求大模型
4. **文档版本控制**：将 `.env.example` 纳入版本控制，`.env` 不要提交
5. **Docker 部署注意**：容器内 Qdrant 使用服务名 `http://qdrant:6333`
