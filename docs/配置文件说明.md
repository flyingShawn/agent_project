# 配置文件完整说明

## 概述

项目采用**三层配置架构**：

| 层级 | 来源 | 优先级 | 说明 |
|------|------|--------|------|
| 1 | `.env` | 最低（全局默认） | 共享配置，所有智能体通用 |
| 2 | `agents.yaml` | 中（智能体覆盖） | 每个智能体独立配置，覆盖 .env 默认值 |
| 3 | `configs/{agent_type}/` | 最高（专属配置） | 提示词、Schema元数据、运维简报等 YAML |

从 `.env.example` 复制后修改：

```bash
copy .env.example .env
```

## 一、智能体总控配置（agents.yaml）

路径：`agent_backend/configs/agents.yaml`

这是多智能体的核心配置，定义了所有启用的智能体及其独立配置。

```yaml
agents:
  - agent_type: desk-agent
    display_name: "桌面管理助手"
    enabled: true
    config_dir: desk-agent
    llm:
      chat_model: "${CHAT_MODEL}"
      vision_model: "${VISION_MODEL}"
    database:
      db_type: mysql
      db_host: "${DB_HOST}"
      db_port: "${DB_PORT}"
      db_name: "${DB_NAME}"
      db_user: "${DB_USER}"
      db_password: "${DB_PASSWORD}"
    rag:
      docs_dir: /data/desk-agent/docs
      docs_collection: desk_agent_docs
      sql_dir: /data/desk-agent/sql
      sql_collection: desk_agent_sql
    reports:
      enabled: true

  - agent_type: ticket-agent
    display_name: "工单系统助手"
    enabled: true
    config_dir: ticket-agent
    llm:
      chat_model: "${CHAT_MODEL}"
    database:
      db_type: mysql
      db_host: "${TICKET_DB_HOST}"
      db_port: "${TICKET_DB_PORT}"
      db_name: "${TICKET_DB_NAME}"
      db_user: "${DB_USER}"
      db_password: "${DB_PASSWORD}"
    rag:
      docs_dir: /data/ticket-agent/docs
      docs_collection: ticket_agent_docs
      sql_dir: /data/ticket-agent/sql
      sql_collection: ticket_agent_sql
    reports:
      enabled: false
```

### 配置项说明

| 字段 | 说明 |
|------|------|
| `agent_type` | 智能体唯一标识，用于 API 路由和数据隔离 |
| `display_name` | 前端显示名称 |
| `enabled` | 是否启用 |
| `config_dir` | 配置子目录名，默认等于 agent_type |
| `llm` | 独立 LLM 配置，支持 `${ENV_VAR}` 引用 .env 变量 |
| `database` | 独立数据库连接，不填则使用 .env 全局默认 |
| `rag` | 独立 RAG 配置（文档目录、集合名） |
| `reports` | 运维简报开关 |

### 新增智能体

只需在 `agents.yaml` 添加一条配置，并在 `configs/` 下创建同名子目录即可：

```bash
mkdir agent_backend/configs/asset-agent
cp agent_backend/configs/desk-agent/prompts.yaml agent_backend/configs/asset-agent/
cp agent_backend/configs/desk-agent/schema_metadata.yaml agent_backend/configs/asset-agent/
```

## 二、大模型配置（.env）

```bash
LLM_BASE_URL=http://localhost:11434/v1
LLM_API_KEY=ollama
CHAT_MODEL=qwen3.5:9b
VISION_MODEL=qwen3.5:9b
```

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `LLM_BASE_URL` | LLM 服务地址（OpenAI 兼容） | `http://localhost:11434/v1` |
| `LLM_API_KEY` | API Key，本地 Ollama 留空 | `ollama` |
| `CHAT_MODEL` | 文本对话模型 | `qwen3.5:9b` |
| `VISION_MODEL` | 视觉模型 | `qwen3.5:9b` |

可在 `agents.yaml` 中为每个智能体覆盖 LLM 配置。

**可选后端**：

| 后端 | LLM_BASE_URL | 需要 API Key |
|------|-------------|-------------|
| 本地 Ollama | `http://localhost:11434/v1` | 否 |
| DeepSeek | `https://api.deepseek.com/v1` | 是 |
| 通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | 是 |

## 三、数据库配置（.env + agents.yaml）

### 全局默认（.env）

```bash
DB_TYPE=mysql
DB_HOST=localhost
DB_PORT=3306
DB_NAME=desk_management
DB_USER=root
DB_PASSWORD=your_password
```

### 智能体独立数据库（agents.yaml）

每个智能体可以配置独立的数据库连接，在 `database` 节点中覆盖：

```yaml
database:
  db_type: mysql
  db_host: "${TICKET_DB_HOST}"
  db_port: "${TICKET_DB_PORT}"
  db_name: "${TICKET_DB_NAME}"
```

对应的 .env 变量：

```bash
TICKET_DB_HOST=192.168.1.100
TICKET_DB_PORT=3306
TICKET_DB_NAME=ticket_system
```

**优先级**：agents.yaml 中的 database 配置 > .env 全局配置

## 四、RAG 配置（agents.yaml）

每个智能体的 RAG 配置在 `agents.yaml` 中定义：

```yaml
rag:
  docs_dir: /data/desk-agent/docs
  docs_collection: desk_agent_docs
  sql_dir: /data/desk-agent/sql
  sql_collection: desk_agent_sql
```

| 字段 | 说明 |
|------|------|
| `docs_dir` | 文档知识库目录路径 |
| `docs_collection` | 文档 Qdrant 集合名 |
| `sql_dir` | SQL 样本目录路径 |
| `sql_collection` | SQL 样本 Qdrant 集合名 |

### Qdrant 连接（.env）

```bash
RAG_QDRANT_URL=http://localhost:6333
RAG_QDRANT_PATH=./.qdrant_local
RAG_QDRANT_API_KEY=
RAG_EMBEDDING_MODEL=BAAI/bge-small-zh-v1.5
```

### 同步命令

```bash
# 同步 desk-agent 文档和 SQL
python scripts/sync.py desk-agent

# 同步 ticket-agent 文档
python scripts/sync.py ticket-agent docs

# 全量同步
python scripts/sync.py desk-agent full

# Docker 模式
.\scripts\sync.cmd desk-agent
```

## 五、智能体专属 YAML 配置

每个智能体在 `configs/{agent_type}/` 下有独立配置目录：

### prompts.yaml

路径：`configs/{agent_type}/prompts.yaml`

定义智能体的提示词：

```yaml
system_prompt: "你是桌面管理助手..."
sql_system_prompt: "你是数据库查询助手..."
sql_prompt_instructions: "..."
summary_prompt: "..."
```

### schema_metadata.yaml

路径：`configs/{agent_type}/schema_metadata.yaml`

定义数据库表结构、同义词、安全规则和查询模板。

### ops_reports.yaml

路径：`configs/{agent_type}/ops_reports.yaml`

定义运维简报配置（仅在 `reports.enabled=true` 时加载）：

```yaml
reports:
  - report_key: default_ops_brief
    enabled: true
    schedule:
      type: daily
      time: "08:00"
      weekday: null
    lookback_days: 3
    top_n: 20
    llm_polish_enabled: true
```

## 六、其他配置（.env）

```bash
CHAT_DB_HOST=localhost
CHAT_DB_PORT=5432
PG_USER=agent
PG_PASSWORD=agent123
PG_DB=agent_chat
CHAT_DB_URL=
CHAT_DB_POOL_SIZE=5
CHAT_DB_MAX_OVERFLOW=10
CHAT_DB_POOL_RECYCLE_SECONDS=3600
CORS_ORIGINS=http://localhost:3000
SQL_MAX_ROWS=500
TAVILY_API_KEY=
WEB_SEARCH_MAX_RESULTS=5
```

| 变量 | 说明 | 默认值 |
|------|------|--------|
| `CHAT_DB_HOST`/`CHAT_DB_PORT` | 聊天历史 PostgreSQL 地址 | `localhost` / `5432` |
| `PG_USER`/`PG_PASSWORD`/`PG_DB` | 聊天历史 PostgreSQL 账号和库名 | `agent` / `agent123` / `agent_chat` |
| `CHAT_DB_URL` | 自定义聊天历史 PostgreSQL 连接URL（留空时自动生成） | 空 |
| `CHAT_DB_POOL_SIZE`/`CHAT_DB_MAX_OVERFLOW` | 聊天历史 PostgreSQL 连接池基础连接数/溢出连接数 | `5` / `10` |
| `CHAT_DB_POOL_RECYCLE_SECONDS` | 聊天历史 PostgreSQL 连接回收秒数 | `3600` |
| `CORS_ORIGINS` | 前端跨域地址 | `http://localhost:3000` |
| `SQL_MAX_ROWS` | SQL 查询最大行数 | `500` |
| `TAVILY_API_KEY` | Tavily 搜索 API Key（留空禁用） | 空 |

## 常见问题

**修改配置后不生效**
1. 确保修改的是 `.env` 文件，而不是 `.env.example`
2. 修改 `agents.yaml` 后重启后端服务

**模型名称错误**
- 查看 Ollama 可用模型：`ollama list`
- 下载模型：`ollama pull qwen3.5:9b`

**数据库连接失败**
1. 数据库服务是否启动
2. 检查 `agents.yaml` 中对应智能体的 `database` 配置
3. 检查 .env 中的环境变量是否正确

**Docker 容器内访问宿主机 Ollama**
- Windows/macOS：`LLM_BASE_URL=http://host.docker.internal:11434/v1`
- Linux：`LLM_BASE_URL=http://172.17.0.1:11434/v1`

## 最佳实践

1. **使用只读数据库账号**：为 AI 智能体创建专用的只读账号
2. **定期备份配置**：备份 `.env` 和 `agents.yaml`
3. **根据硬件选择模型**：不要盲目追求大模型
4. **新增智能体只需修改配置**：在 `agents.yaml` 添加条目 + 在 `configs/` 创建子目录
5. **Docker 部署注意**：容器内 Qdrant 使用服务名 `http://qdrant:6333`
