## 🚀 测试步骤

### 第零步：检查服务是否已启动

**方法1：检查端口占用**
```bash
netstat -ano | findstr :8000
```

**预期输出**：
```
TCP    0.0.0.0:8000    0.0.0.0:0    LISTENING    <进程ID>
```

如果看到类似输出，说明服务已经在运行。

**方法2：测试健康检查**
```bash
python -c "import urllib.request; print(urllib.request.urlopen('http://localhost:8000/api/v1/health').read().decode())"
```

**预期结果**：
```json
{"status":"ok"}
```

如果返回 `{"status":"ok"}`，说明服务已经正常运行，可以跳过第一步。

### 服务关闭方法

#### 关闭后端服务

```bash
# 1. 查找进程ID
netstat -ano | findstr :8000

# 2. 关闭进程（替换12345为实际进程ID）
taskkill /PID 12345 /F
```

#### 关闭Ollama服务

```bash
# 方法1：根据进程ID关闭
netstat -ano | findstr :11434
taskkill /PID <进程ID> /F

# 方法2：根据进程名称关闭
taskkill /IM ollama.exe /F
```

#### 关闭Qdrant服务

```bash
# 如果是Docker方式
docker stop desk-agent-qdrant

# 如果是进程方式
netstat -ano | findstr :6333
taskkill /PID <进程ID> /F
```

---

### 服务重启方法

#### 重启后端服务

```bash
# 1. 关闭现有服务
netstat -ano | findstr :8000
taskkill /PID <进程ID> /F

# 2. 重新启动
python -m uvicorn agent_backend.main:app --reload --host 0.0.0.0 --port 8000
```

#### 重启Ollama服务

```bash
# 1. 关闭现有服务
taskkill /IM ollama.exe /F

# 2. 重新启动
ollama serve
```

#### 重启Qdrant服务

```bash
# Docker方式
docker restart desk-agent-qdrant
```

## 🔧 常见问题

### 问题1：端口被占用

**错误信息**：`Address already in use`

**解决方案**：
```bash
# 查找占用端口的进程
netstat -ano | findstr :8000

# 结束进程（替换PID）
taskkill /PID <进程ID> /F
```

### 问题2：Ollama连接失败

**错误信息**：`调用 Ollama 失败`

**解决方案**：
```bash
# 检查Ollama服务
ollama list

# 如果没有运行，启动Ollama
ollama serve
```

### 问题3：模型不存在

**错误信息**：`model not found`

**解决方案**：
```bash
# 下载模型
ollama pull qwen2.5:7b-instruct
```

### 问题4：数据库连接失败

**错误信息**：`Access denied for user`

**解决方案**：
1. 检查 `.env` 文件中的数据库配置
2. 确认数据库服务已启动
3. 验证用户名和密码

### 问题5：npm命令不存在

**错误信息**：`npm : 无法将"npm"项识别为 cmdlet、函数、脚本文件或可运行程序的名称`

**原因**：未安装 Node.js 和 npm

**解决方案**：

**方法1：官网下载安装（推荐）**
1. 访问 Node.js 官网：https://nodejs.org/
2. 下载 LTS 版本（推荐 20.x 或 22.x）
3. 运行安装包，保持默认选项
4. 安装完成后，**关闭并重新打开命令行窗口**

**方法2：使用 winget 安装**
```bash
winget install OpenJS.NodeJS.LTS
```

**验证安装**：
```bash
# 检查 Node.js 版本
node --version

# 检查 npm 版本
npm --version
```

**详细安装指南**：查看 [help/Node.js安装指南.md](help/Node.js安装指南.md)

---

### 问题6：前端依赖安装失败

**错误信息**：`npm install` 失败

**解决方案**：
```bash
# 清除 npm 缓存
npm cache clean --force

# 删除 node_modules 目录
rmdir /s /q node_modules

# 重新安装依赖
npm install
```
4. 检查用户权限
