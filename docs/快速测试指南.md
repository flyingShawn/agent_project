# 快速测试指南

## 测试目标

验证多智能体系统的核心功能是否正常运行。

---

## 前置条件

| 服务 | 端口 | 检查方式 |
|------|------|----------|
| Ollama | 11434 | `ollama list` |
| Qdrant | 6333 | 浏览器访问 `http://localhost:6333/dashboard` |
| MySQL/PostgreSQL | 3306/5432 | `python scripts/测试数据库连接.py` |

---

## 启动步骤

### 1. 启动后端

```bash
python -m uvicorn agent_backend.main:app --host 0.0.0.0 --port 8000
```

验证：`http://localhost:8000/api/v1/health` 返回 `{"status":"ok"}`

### 2. 启动前端

```bash
cd agent_frontend
npm install   # 首次需要
npm run dev
```

验证：浏览器访问 `http://localhost:3000`

## 功能测试

### 智能体列表

```bash
python -c "import urllib.request, json; print(json.loads(urllib.request.urlopen('http://localhost:8000/api/v1/agents').read().decode()))"
```

### desk-agent 测试

在前端访问 `http://localhost:3000/desk-agent`，输入：
- "查看客户端在线状态"
- "当前有多少部门"

### ticket-agent 测试

在前端访问 `http://localhost:3000/ticket-agent`，输入：
- "最近一周有多少工单"
- "查询未关闭的工单"

### 数据隔离验证

1. 在 desk-agent 发送消息，确认历史记录出现
2. 切换到 ticket-agent，确认历史记录独立（不显示 desk-agent 的记录）

### 运维简报

```bash
.\scripts\ops_report.cmd desk-agent          # 手动触发
.\scripts\ops_report.cmd desk-agent latest   # 查看最新
.\scripts\ops_report.cmd desk-agent list     # 列表
```

### 文档/SQL 同步

```bash
python scripts/sync.py desk-agent            # 同步 desk-agent
python scripts/sync.py ticket-agent docs     # 同步 ticket-agent 文档
python scripts/sync.py desk-agent sql full   # 全量同步 SQL
```

## 常见问题

| 问题 | 解决方案 |
|------|----------|
| 端口 8000 被占用 | `Get-NetTCPConnection -LocalPort 8000 \| Select-Object -ExpandProperty OwningProcess \| ForEach-Object { Stop-Process -Id $_ -Force }` |
| Ollama 连接失败 | 检查 `ollama list` 是否有模型，运行 `ollama serve` |
| 模型不存在 | `ollama pull qwen3.5:9b` |
| 数据库连接失败 | 检查 `agents.yaml` 中对应智能体的 `database` 配置 |
| npm 命令不存在 | 安装 Node.js：https://nodejs.org/ |
| PowerShell 脚本限制 | `Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass` |
