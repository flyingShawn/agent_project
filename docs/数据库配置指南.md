# 数据库连接配置指南

## 📍 配置位置

数据库连接配置在项目根目录的 **`.env`** 文件中。

**配置方式**：支持两种配置方法，推荐使用分离配置。

---

## 🔧 配置方法

### 方法一：分离配置（推荐）

**优点**：清晰易懂，自动处理特殊字符

```bash
# 数据库类型
DB_TYPE=mysql

# 连接信息
DB_HOST=localhost
DB_PORT=3306
DB_NAME=desk_management
DB_USER=root
DB_PASSWORD=your_password
```

### 方法二：完整URL配置

**优点**：灵活，支持更多参数

```bash
DATABASE_URL=mysql+pymysql://root:password@localhost:3306/db?charset=utf8mb4
```

**优先级**：如果同时配置，`DATABASE_URL` 优先

---

## 📝 MySQL 配置示例

### 本地MySQL（无密码）
```bash
DB_TYPE=mysql
DB_HOST=localhost
DB_PORT=3306
DB_NAME=desk_management
DB_USER=root
DB_PASSWORD=
```

### 本地MySQL（有密码）
```bash
DB_TYPE=mysql
DB_HOST=localhost
DB_PORT=3306
DB_NAME=desk_management
DB_USER=root
DB_PASSWORD=123456
```

### 远程MySQL服务器
```bash
DB_TYPE=mysql
DB_HOST=192.168.1.100
DB_PORT=3306
DB_NAME=desk_management
DB_USER=admin
DB_PASSWORD=SecurePass123
```

### 使用完整URL
```bash
DATABASE_URL=mysql+pymysql://root:password@localhost:3306/desk_management?charset=utf8mb4
```

## 🔒 安全建议

### 创建只读账号（强烈推荐）

#### MySQL创建只读用户：
```sql
-- 创建只读用户
CREATE USER 'ai_readonly'@'%' IDENTIFIED BY 'your_secure_password';

-- 授予只读权限
GRANT SELECT ON desk_management.* TO 'ai_readonly'@'%';

-- 刷新权限
FLUSH PRIVILEGES;
```

然后修改 `.env` 文件：
```bash
DB_USER=ai_readonly
DB_PASSWORD=your_secure_password
```

#### PostgreSQL创建只读用户：
```sql
-- 创建只读用户
CREATE USER ai_readonly WITH PASSWORD 'your_secure_password';

-- 授予连接权限
GRANT CONNECT ON DATABASE desk_management TO ai_readonly;

-- 授予只读权限
GRANT USAGE ON SCHEMA public TO ai_readonly;
GRANT SELECT ON ALL TABLES IN SCHEMA public TO ai_readonly;
```

## 📋 数据库权限要求

AI智能体只需要以下权限：

### MySQL权限：
- `SELECT` - 查询数据

### PostgreSQL权限：
- `CONNECT` - 连接数据库
- `USAGE` - 使用Schema
- `SELECT` - 查询数据

**不需要的权限**：
- ❌ INSERT（插入）
- ❌ UPDATE（更新）
- ❌ DELETE（删除）
- ❌ CREATE（创建）
- ❌ DROP（删除表）
- ❌ ALTER（修改结构）

## ⚠️ 常见问题

### 问题1：连接失败 - Access denied

**错误信息**：`Access denied for user 'root'@'localhost'`

**解决方案**：
1. 检查用户名和密码是否正确
2. 检查数据库用户是否有远程访问权限
3. 运行配置检查：`检查配置.bat`

### 问题2：连接失败 - Unknown database

**错误信息**：`Unknown database 'desk_management'`

**解决方案**：
1. 确认数据库名称是否正确
2. 确认数据库已创建
3. 检查用户是否有访问该数据库的权限

### 问题3：连接超时

**错误信息**：`Can't connect to MySQL server on 'xxx'`

**解决方案**：
1. 检查数据库服务是否启动
2. 检查防火墙是否开放端口（MySQL默认3306，PostgreSQL默认5432）
3. 检查IP地址和端口是否正确
4. 尝试使用 `ping` 命令测试网络连通性

### 问题4：字符编码问题

**错误信息**：中文显示乱码

**解决方案**：
- MySQL：在URL中添加 `?charset=utf8mb4`（分离配置会自动添加）
- PostgreSQL：确保数据库编码为 `UTF8`
