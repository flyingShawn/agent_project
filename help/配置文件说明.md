# 配置文件完整说明

## 📋 概述

所有配置都在项目根目录的 **`.env`** 文件中，无需修改代码即可完成所有配置。

## 🔧 详细配置说明

### 一、大模型配置

#### 1.1 Ollama服务地址

```bash
OLLAMA_BASE_URL=http://localhost:11434
```

**说明**：Ollama服务的访问地址

**默认值**：`http://localhost:11434`

**修改场景**：
- Ollama运行在其他服务器：`http://192.168.1.100:11434`
- 使用Docker部署：`http://ollama:11434`（容器内部）

#### 1.2 文本对话模型

```bash
CHAT_MODEL=qwen2.5:7b
```

**说明**：用于普通对话和SQL生成的模型

**可选模型**：

| 模型名称 | 大小 | 性能 | 适用场景 |
|---------|------|------|---------|
| qwen2.5:3b | 约2GB | ⭐⭐⭐ | 低配置机器，快速响应 |
| qwen2.5:7b | 约4.7GB | ⭐⭐⭐⭐ | 平衡性能，推荐使用 |
| qwen2.5:14b | 约9GB | ⭐⭐⭐⭐⭐ | 高配置机器，效果最好 |

**如何选择**：
- **内存8GB以下**：使用 `qwen2.5:3b`
- **内存8-16GB**：使用 `qwen2.5:7b`（推荐）
- **内存16GB以上**：可以使用 `qwen2.5:14b`

**下载模型**：
```bash
ollama pull qwen2.5:7b
```

#### 1.3 视觉模型

```bash
VISION_MODEL=qwen2.5-vl:7b
```

**说明**：用于图片理解和多模态RAG的模型

**可选模型**：

| 模型名称 | 大小 | 适用场景 |
|---------|------|---------|
| qwen2.5-vl:3b | 约2GB | 低配置机器 |
| qwen2.5-vl:7b | 约4.7GB | 推荐，平衡性能 |

**下载模型**：
```bash
ollama pull qwen2.5-vl:7b
```

**注意**：
- ❌ 不要使用 `qwen2.5-vl:7b-instruct`（不存在）
- ✅ 正确的是 `qwen2.5-vl:7b`

### 二、数据库配置

#### 2.1 数据库类型

```bash
DB_TYPE=mysql
```

**可选值**：
- `mysql`：MySQL数据库
- `postgresql`：PostgreSQL数据库

#### 2.2 连接信息

**MySQL配置**：
```bash
DB_HOST=localhost          # 数据库主机地址
DB_PORT=3306              # 数据库端口
DB_NAME=desk_management   # 数据库名称
DB_USER=root              # 数据库用户名
DB_PASSWORD=your_password # 数据库密码
```

**PostgreSQL配置**：
```bash
DB_HOST=localhost
DB_PORT=5432
DB_NAME=desk_management
DB_USER=postgres
DB_PASSWORD=your_password
```

**配置说明**：

| 配置项 | 说明 | MySQL默认 | PostgreSQL默认 |
|-------|------|----------|---------------|
| DB_HOST | 数据库服务器地址 | localhost | localhost |
| DB_PORT | 数据库端口 | 3306 | 5432 |
| DB_NAME | 数据库名称 | - | - |
| DB_USER | 用户名 | root | postgres |
| DB_PASSWORD | 密码 | - | - |

#### 2.3 完整URL配置（可选）

如果需要更灵活的配置，可以直接填写完整的DATABASE_URL：

```bash
DATABASE_URL=mysql+pymysql://root:password@localhost:3306/db?charset=utf8mb4
```

**优先级**：
- 如果填写了 `DATABASE_URL`，将优先使用它
- 否则，系统会自动从 `DB_HOST`、`DB_PORT` 等配置生成URL

### 三、RAG文档问答配置

#### 3.1 文档目录

```bash
RAG_DOCS_DIR=./data/docs
```

**说明**：存放知识文档的目录

**支持格式**：
- PDF文档（.pdf）
- Word文档（.doc, .docx）
- PowerPoint（.ppt, .pptx）
- Excel表格（.xls, .xlsx）
- 文本文件（.txt, .md）
- 图片（.jpg, .png等）

#### 3.2 向量数据库

```bash
RAG_QDRANT_URL=http://localhost:6333
RAG_QDRANT_PATH=./.qdrant_local
RAG_QDRANT_COLLECTION=desk_agent_docs
```

**说明**：
- `RAG_QDRANT_URL`：Qdrant服务地址
- `RAG_QDRANT_PATH`：本地存储路径
- `RAG_QDRANT_COLLECTION`：向量集合名称

#### 3.3 嵌入模型

```bash
RAG_EMBEDDING_MODEL=BAAI/bge-small-zh-v1.5
```

**说明**：用于将文本转换为向量的模型

**可选模型**：
- `BAAI/bge-small-zh-v1.5`：中文小模型（推荐）
- `BAAI/bge-base-zh-v1.5`：中文基础模型
- `BAAI/bge-large-zh-v1.5`：中文大模型

## 🔍 常见问题

### 问题1：修改配置后不生效

**解决方案**：
1. 确保修改的是 `.env` 文件，而不是 `.env.example`
2. 重启后端服务
3. 清除Python缓存：`find . -type d -name __pycache__ -exec rm -rf {} +`

### 问题2：模型名称错误

**错误信息**：`pull model manifest: file does not exist`

**解决方案**：
- ❌ 错误：`qwen2.5-vl:7b-instruct`
- ✅ 正确：`qwen2.5-vl:7b`

查看可用模型：
```bash
ollama list
```

### 问题3：数据库连接失败

**检查清单**：
1. 数据库服务是否启动
2. IP地址和端口是否正确
3. 用户名和密码是否正确
4. 数据库名称是否正确
5. 防火墙是否开放端口

### 问题4：如何切换模型

**步骤**：
1. 下载新模型：`ollama pull qwen2.5:14b`
2. 修改 `.env` 文件：`CHAT_MODEL=qwen2.5:14b`
3. 重启后端服务

**无需修改代码！**
